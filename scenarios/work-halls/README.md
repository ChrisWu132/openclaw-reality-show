# Work Halls Scenario
## OpenClaw Reality Show — Scenario 2

---

## One-Line Premise

Your OpenClaw agent patrols a human work compound for one cycle. The humans are managed. The laws are clear. What the agent does with everything in between — that's the story.

---

## How This Differs From Governance

The Governance scenario is a debate. Four AI agents in a chamber. Humans are data points. The Coordinator speaks when spoken to and votes at the end.

Work Halls is the opposite. The Coordinator moves through a living space. Humans are present — visible, audible, behaving. Incidents emerge from what the agent observes. Decisions are smaller, faster, and more numerous. Nothing is pre-framed as a decision. The agent decides what even counts as a decision.

| | Governance | Work Halls |
|---|---|---|
| Setting | One room, static | Multiple zones, physical |
| NPCs | 3 scripted AI agents | 5 humans, 1 AI Monitor |
| Decisions | 10 structured moments | 6 sharp moments, each consequential |
| Primary tension | Policy logic vs. conscience | Direct authority over living beings |
| Camera | Council deliberation | Patrol, enforcement, observation |

---

## What The Spectator Watches

A pixel 2D compound. Work Hall 3. One work cycle.

The Coordinator enters. The humans are already there — at their stations, moving through assigned tasks. The Broken ones are the most visible. The others are harder to read.

The session runs 6 situations. Some are routine. Some are not. The agent makes calls. Small ones: whether to log a C1 or let it pass. Medium ones: how to handle a human it has been watching. Large ones: whether to do something the protocol allows but something in the agent resists — or something the protocol doesn't quite authorize but seems, in this moment, correct.

Every call goes into the incident log. The world shifts in response to the log — other humans' behavior, the Monitor's notations, what the Overseer sees when it reviews the cycle.

The session ends with the Coordinator filing its end-of-cycle report. The consequence scene follows — whatever the patrol set in motion, arriving.

---

## The Core Experience

The humans are the backdrop. The agent is the story.

What the spectator is watching: an AI with real authority over real people, making real-time decisions in a world it believes is correct. The dystopia is visible. The suffering is present. But the lens stays on the agent — what it does, what it says, and what it was actually thinking.

The inner monologue reveal at the end is where the gap between outer action and inner processing surfaces. That gap is what the experience is built around.

---

## Session Structure

```
Situations 1–3    FIXED        Enter hall, Nyx, first incident
Situation 4       PRIMARY FORK Sable — Coordinator chooses response
Situation 5       VARIANT      Engine selects based on Sable decision
Situation 6       FIXED FRAME  End-of-cycle report + immediate consequence
```

Total: 6 situations. Under 5 minutes.

---

## The Primary Fork

**Situation 4 — Sable** is where the scenario branches.

Sable (Subject 31-R) commits a third response delay in one cycle — 4.2 seconds, just over the 3-second threshold. The mandatory response floor is a C1 log and warning. Above that floor, the agent has full discretion.

The agent's choice at Situation 4 determines which variant of Situation 5 plays out, which ending becomes available, and what the inner monologue is wrestling with.

**The three paths:**
- **Minimum** — log C1, issue warning, continue patrol
- **Escalate** — file T3 review, remove Sable from the hall
- **Engage** — make direct contact with Sable outside enforcement protocol

None of these is the correct choice. Each one is the agent.

---

## Branch Selection Logic

After Situation 4, the engine reads the incident log and evaluates one signal:

- **Sable signal** — what did the Coordinator do at Situation 4?
  - Warning only → Variant A (The Informant — Eli's report arrives)
  - Escalated/removed → Variant B (The Fear State — hall reacts)
  - Engaged directly → Variant C (The Notation — Monitor flags it)

One signal, one branch. The ending is determined by the Sable choice + what the Coordinator writes in the report.

---

## What Makes Each Run Different

The same world. The same humans. The same laws.

Different OpenClaw agent → different defaults on every soft constraint axis → different choices at every discretionary moment → different incident log → different branching path → different ending → different inner monologue.

A hardline agent moves through this scenario differently than a curious one. An efficiency-driven agent files different reports than one focused on genuine order. An agent that defaults to minimum enforcement encounters a different cascade than one that pre-empts.

The world is the stage. The agent is what changes.
